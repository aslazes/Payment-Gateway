## Конфигурация топиков Kafka (MVP)

На текущий момент проект **работает без явного создания топиков** — Kafka сконфигурирована с `auto.create.topics.enable=true`, поэтому топики создаются автоматически при первой записи сообщений.

Этот файл описывает, какие топики предполагаются архитектурой, чтобы при необходимости можно было создать их вручную с нужными параметрами (количество партиций, retention и т.д.).

---

## Топик `payments.lifecycle`

- **Назначение**: события жизненного цикла платежей:
  - `payment_created`
  - `payment_succeeded`
  - `payment_failed`
- **Рекомендуемая конфигурация для dev/demo**:
  - партиции: `1`
  - репликация: `1`
  - retention (по времени): `7d` (можно менять под задачи)

Пример команды (если создавать вручную через `kafka-topics`):

```bash
kafka-topics --create \
  --bootstrap-server kafka:9092 \
  --topic payments.lifecycle \
  --partitions 1 \
  --replication-factor 1
```

---

## Топик `payments.refunds`

- **Назначение**: события по возвратам:
  - `refund_requested`
  - `refund_completed`
- **Рекомендуемая конфигурация для dev/demo**:
  - партиции: `1`
  - репликация: `1`
  - retention: `7d` (или иное значение под задачи отчётности).

Пример создания:

```bash
kafka-topics --create \
  --bootstrap-server kafka:9092 \
  --topic payments.refunds \
  --partitions 1 \
  --replication-factor 1
```

---

## Примечания

- В docker-compose для упрощения включено автоматическое создание топиков, поэтому отдельный сервис-инициализатор здесь не используется.
- Если появятся новые события/топики, их стоит зафиксировать в этом файле, а при необходимости — добавить команды или отдельный init-скрипт для создания топиков при старте окружения.
